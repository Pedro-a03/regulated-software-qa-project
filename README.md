## About This Project

This repository demonstrates a **senior-level QA approach to regulated software**, combining formal test planning aligned with ISO 13485 / EU MDR / IEC 62304 with modern API test automation using Python, pytest, and a CI/CD pipeline via GitHub Actions.

**System Under Test:** *QualityHub* — a fictional Electronic Quality Management System (eQMS) for regulated industries. The software manages CAPA records, internal audit action items, and controlled document workflows — core processes required by ISO 13485:2016 and EU MDR 2017/745.

**Why eQMS?**
eQMS platforms (such as Veeva Vault, Greenlight Guru, MasterControl) are widely used by medical device manufacturers, pharma companies, and precision manufacturing firms — the exact industries where this QA profile operates. Understanding how to test these systems from both a regulatory and technical perspective is a key differentiator.

**What makes this project different:**
Junior QA profiles can test features. This project tests *compliance requirements* — audit trail immutability, RBAC enforcement, document ownership traceability — framing every test case against the regulatory clause it validates.
